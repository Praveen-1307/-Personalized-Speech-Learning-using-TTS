# Implementation Specification: Emotion Detector (`emotion_detector.py`)

## 1. Overview
The Emotion Detector acts as the "Psychological Vibe Checker." Its purpose is to predict if a user sounds happy, sad, angry, or calm based on a recording.

## 2. The Problem It Solves
A computer does not understand language tone directly. This module translates the physical acoustic shapes of a user's voice into categorical emotional states, allowing the system to adjust synthesis parameters accordingly.

## 3. How It Works in the Code 
1. **Model State Check:** By default, the detector acts as a passive mock. Unless an active SVM model is explicitly trained or loaded into it, it will always safely default to predicting a "neutral" state with 100% confidence rather than crashing.
2. **Classical Machine Learning:** When activated, it uses a Support Vector Machine (SVM) or RandomForest, rather than a massive Deep Learning neural network.
3. **Modeling the Throat:** The code looks at mathematical mappings (MFCCs, spectral centroids, pitch variation) generated by librosa.
4. **Classification:** It feeds these numbers into the SVM. If the SVM recognizes that the numbers look like "Shape A", it knows the person is yelling and outputs the string "angry". 

## 4. Technical Architecture

### 4.1 Inputs
- **Raw Audio:** 1D numpy array comprising the input vocal sample.
- **Sample Rate:** Integer dictating the recording speed.

### 4.2 Execution Flow
1. Verify if the internal SVM/RandomForest model exists and `is_trained == True`.
2. If untrained, safely bypass processing and return a default 'neutral' dictionary.
3. If trained, extract specific audio features using `librosa`.
4. Scale features using the fitted `StandardScaler`.
5. Run the classifier's `predict_proba` function to yield the best match.

### 4.3 Data Transformations
- **Audio to Features:** Compresses continuous amplitude time-series into a condensed 1D Array of mean statistical features (MFCCs, zerocrossing, F0 variations).
- **Features to Scaled Tensors:** Normalizes array means locally via Z-score standard scaling.
- **Scaled Tensors to Class Logits:** Converts the mathematical input into probabilistic classification categories.

### 4.4 Outputs
- **Result Dictionary:** Contains keys `emotion` (string label), `confidence` (float percentage), and `probabilities` (full dictionary of mapped odds).

### 4.5 Current Limitations
- **Untrained Default:** Currently defaults to 'neutral' because a pre-trained `.pkl` model does not ship with the repository. Needs active training sequence first.
- **Context Ignorant:** It only evaluates acoustic data. If a user smiles while recounting a sad story, the acoustics may conflict with semantic truth.
